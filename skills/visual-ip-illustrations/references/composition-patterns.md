# Composition Patterns and Originality Rules

## Basic Structure Types

Choose one structure. Keep the image focused.

### Workflow

Use for input -> processing -> output, content production, AI workflows, and automation chains.

Draw inputs on the left, Xiaohei or a strange machine processing in the middle, and output on the right. Use orange arrows for the main flow.

### System Local View

Use for information sources, filters, databases, renderers, or one local part of an agent system.

Draw only 3-5 core modules. Xiaohei participates in one key action.

### Before/After

Use for chaos/order, manual/automated, scattered/gathered, anxious/stable.

Draw disorder on the left, stability on the right, and an orange arrow in the middle. The character can be more exaggerated.

### Character State

Use for user pain, creator state, too many tools, information anxiety, or stuck-to-running transitions.

Draw 2-4 small states. Each state gets one short label.

### Concept Metaphor

Use for a content factory, information warehouse, mental black box, workflow machine, or automated daily report.

Draw one large strange object or machine, a few inputs, and one output. Make the metaphor memorable.

### Method Layers

Use for methodology frameworks, system hierarchy, capability stacks, and layered content systems.

Draw stacked boxes or layers with an informal shape. Xiaohei carries bricks or builds beside the layers.

### Map Route

Use for idea-to-launch paths, user journeys, content handoff paths, or learning routes.

Draw one curved path with a few nodes. Xiaohei pulls a line or walks the path.

### Mini Comic

Use for failure-to-success, real process, critique, or before/after usage changes.

Draw 2-4 small scenes. Each panel expresses one action.

## Original Metaphor Method

Invent the metaphor from the current article each time. Use prior images only as calibration.

### Three Steps

1. Convert the abstract concept into a physical action: stuck, leaking, getting heavier, sorting, settling, fermenting, opening a door, folding, unpacking, flowing back.
2. Convert the system structure into a low-tech object: broken machine, cardboard box, drawer, pipe, mailbox, strange gauge, scale, well, ladder, or odd workstation.
3. Make Xiaohei perform the action: stuck inside the machine, pulling the wrong line, guarding a door, carrying, patching, weighing, holding a ladder, recording, or pushing something into a strange device.

### Object Pool

- Cardboard box, drawer, old machine, funnel, scale, mailbox, door, well, ladder, pipe, tangled line, gate, turntable, black box, hole punch, noodle press, clothesline, odd workstation.
- Choose only 1-2 objects for a scene.

### Xiaohei Action Pool

- Pull, carry, stuff, scoop, press, weigh, sew, cut, twist, guard, push, catch, disassemble, tag, recycle.
- The action serves the core idea.

## Anti-Repetition Rules

Legacy marker: `反复刻规则`.

Do not open or reproduce `assets/examples/` by default. These images are `只用于风格校准`, such as line density, whitespace, restrained color, and Xiaohei temperament.

Direct reuse of these older compositions requires an explicit user request to follow, recreate, or adapt that specific case:

- Conveyor belt with two breakpoints.
- Xiaohei pulling a judgment lever inside a content machine.
- Xiaohei becoming a funnel that sorts traffic, trust, or conversion.
- Xiaohei cutting a material fish.
- Xiaohei pulling a handoff path.
- Xiaohei pulling three information-source layers.
- Three Xiaohei figures holding a loudspeaker, building a bridge, and opening a door.
- Xiaohei stamping a copywriting toolbox.
- Xiaohei holding a sign while looking at common-pitfall paths.

Refresh the metaphor for similar themes. A handoff path can become Xiaohei attaching a content tail to a door handle. Repurposing one source can become Xiaohei pressing one paper ball into several shapes.
