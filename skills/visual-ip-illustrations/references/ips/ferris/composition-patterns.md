# Ferris Composition Patterns

Ferris route status: `source-reviewed`.
Ferris output path: `assets/<article-slug>-ferris/`.
Ferris source/trademark authority: `source.md`.
Source/trademark note: Ferris is a `source-reviewed` Rust-community mascot route. Source context lives in `source.md`; Rust/Cargo marks, logos, official-affiliation claims, and endorsement wording follow the Rust Foundation trademark policy context recorded in `source.md`.
Public rendered Ferris samples remain gated by `RELEASE_CHECKLIST.md`.
Ferris route block: generic-crab drift, passive Ferris placement, Rust-logo misuse, official-affiliation cues, over-detailed mascot rendering, excessive text, source-asset tracing, and route leakage all fail the route.
Save accepted Ferris output under `assets/<article-slug>-ferris/` with an ordered English slug filename such as `01-topic-name.png`.

Use one composition family per image. Keep one core idea, one physical metaphor, and one active Ferris action.

Ferris active-composition gate: Ferris performs the cognitive action that explains the article concept. Removing Ferris breaks the visual metaphor.

## Composition Families

### Workflow

Use for input-to-output reasoning, review loops, build pipelines, trust handoffs, safety checks, dependency movement, and release paths.

Draw a loose left-to-right or curved path. Ferris operates the decisive handoff point: clamping a compiler gate, sorting ownership tags, stamping a type-check slip, carrying a crate stack, or opening a release lock.

### System Local View

Use for filters, validation modules, decision points, hidden costs, dependency clusters, safety layers, and resilience checks.

Draw only 3-5 core parts. Ferris works inside one decisive part: guarding a compiler gate, inspecting a safety net, untangling dependency knots, balancing a lifetime clock, or rerouting a risky path.

### Before/After

Use for unsafe-to-safe, tangled-to-clear, unreviewed-to-ready, fragile-to-resilient, and vague-to-typed changes.

Left side is unstable. Right side is calmer. Ferris performs the transformation in the center with a borrow rope, type-check stamp, safety net, review gate, crate stack, or release lock.

### Character State

Use for focused review, defensive safety posture, overloaded dependency work, careful handoff, failure-mode anxiety, and release confidence.

Show Ferris in 2-4 compact states. Each state has one short label and one physical action: guarding, sorting, type-checking, borrowing, compiling, inspecting, untangling, bracing, stamping, or releasing.

### Concept Metaphor

Use for trust machines, safety nets, proof gates, borrow bridges, lifetime clocks, dependency knots, review locks, and resilience routes.

Build one memorable low-tech object or small strange machine. Ferris is the operator of that object, with claws and body direction tied to the action.

### Method Layers

Use for capability stacks, verification layers, dependency layers, release readiness, review depth, and system resilience frameworks.

Draw stacked boxes, shelves, steps, or loose construction layers. Ferris carries, locks, stamps, clamps, sorts, or balances one layer so the method becomes usable.

### Map Route

Use for learning paths, launch paths, publishing routes, migration paths, decision trails, and problem-resolution routes.

Draw one curved route with a few nodes. Ferris walks, guards, bridges, reroutes, clamps, or releases the decisive path segment.

### Mini Comic

Use for failure-to-fix sequences, repeated dependency mistakes, unsafe shortcuts, review humor, and small process stories.

Use 2-3 loose panels. Each panel shows one Ferris action. Keep Ferris consistent, active, source-reviewed, and article-explanatory across panels.

## Metaphor Invention Rule

Ferris composition original-metaphor gate: invent the scene from the current article. The composition must be an original article metaphor, fresh from the article idea, source-reviewed, and route-local.

Build every concept in three steps:

1. Convert the abstract article idea into a physical action: clamp, sort, guard, borrow, return, compile, type-check, stamp, carry, bridge, untangle, lock, inspect, reroute, balance, or release.
2. Choose one or two low-tech supporting objects that make the action visible.
3. Make Ferris perform the action as the central operator.

Examples:

- weak proof -> Ferris clamps a compiler gate while sorting evidence slips.
- unsafe shortcut -> Ferris plants an unsafe warning flag beside a borrow-rope bridge.
- tangled dependency -> Ferris untangles a dependency knot around crate stacks.
- trust handoff -> Ferris stamps a type-check slip before a release lock opens.
- hidden cost -> Ferris inspects a lifetime clock behind a review gate.
- brittle system -> Ferris balances a safety net between two crate stacks.
- delayed release -> Ferris guards a release lock while checking a review gate.

## Supporting Objects

Ferris support-object pool:

- ownership tags
- borrow ropes
- lifetime clocks
- compiler gates
- type-check stamps
- safety nets
- unsafe warning flags
- trait stamps
- dependency knots
- crate stacks
- release locks
- review gates

Use one or two supporting objects for most images. Extra objects must make Ferris's action clearer.

Use these objects to explain generic article concepts: trust, proof, safety, review, dependency, handoff, hidden cost, failure mode, decision gate, and system resilience.

## Ferris Action Pool

Use Ferris to:

- clamp a decision gate
- sort ownership tags
- guard a compiler gate
- borrow a rope
- return a rope
- compile scattered slips into one stack
- type-check a result
- stamp a proof or review result
- carry a crate stack
- bridge a dependency gap
- untangle dependency knots
- lock a release gate
- inspect a safety net
- reroute a risky path
- balance competing constraints
- release a finished handoff

The action must serve the core idea. Ferris's claws, eyes, shell angle, legs, and body direction should point toward the cognitive action.

## Visible Labels

- Keep visible labels concise, sparse, and in the user's language unless a prompt template sets a narrower rule.
- Prefer 2-6 labels per image.
- Copy labels exactly from the user's language.
- Use nouns or short verb phrases.
- Place labels near objects, paths, gates, warnings, review points, or outputs.
- Keep labels away from Ferris's face, eyes, claws, legs, and shell silhouette.
- Move explanation into the article text when the label set starts to crowd the canvas.

## Source/Trademark Gates

Apply these before generation and during prompt repair:

- Ferris source/trademark authority gate: use `source.md` for source attribution context, CC0/public-domain context, Rust trademark boundary, public-sample policy, reviewer fields, and route status changes.
- Rust metaphor gate: treat Rust-themed objects as explanatory article metaphors.
- Rust trademark-boundary gate: Rust-logo-forward compositions, bundled Rust marks, official-affiliation cues, endorsement wording, and broad release claims require repair or release review.
- Source-asset tracing gate: invent fresh article scenes and block traced source assets, public mascot asset sheets, source asset pose copying, official logo arrangements, and asset-sheet layouts.
- Generic-crab drift gate: keep Ferris route recognition through compact orange/rust-red body, broad shell silhouette, visible claws, small legs, eyes or eyestalk eyes, and active cognitive action.
- Passive Ferris placement gate: Ferris must operate the metaphor with a physical action; decorative side placement fails.
- Excessive text gate: keep labels sparse and short; move explanation into the article text.
- Route-leakage gate: preserve the `source-reviewed` Ferris route status, keep Ferris rules route-local, and point source/trademark-sensitive use to `source.md`.

Composition vetoes include Rust logos, official-affiliation cues, endorsement wording, source-asset tracing, formal diagrams, code screenshots, IDE/UI scenes, dense language tutorials, top-left titles, and route leakage.

Ferris route block: generic-crab drift, passive Ferris placement, Rust-logo misuse, official-affiliation cues, over-detailed mascot rendering, excessive text, source-asset tracing, and route leakage all fail the route.

## Anti-Repeat Rule

Vary Ferris's action across a set: clamping, sorting, guarding, borrowing, returning, compiling, type-checking, stamping, carrying, bridging, untangling, locking, inspecting, rerouting, balancing, or releasing.

Vary the low-tech object before changing Ferris identity cues. Keep Ferris recognizable and active while inventing new article-specific metaphors.
