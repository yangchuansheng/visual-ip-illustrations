# Littlebox Composition Patterns

Choose one composition family per image. Keep one core idea and one Littlebox action.

## Intake And Packaging

Use for raw notes, messy content, idea capture, research input, and knowledge organization.

Draw closed Littlebox on one side of the canvas using side-seam twig arms to gather loose slips, blocks, or signals into an external stack, sealed bundle, or labeled packet. The lid stays closed.

## Before/After

Use for chaos-to-structure, manual-to-system, vague-to-clear, scattered-to-packaged.

Left side is loose and unstable. Right side is compact and calm. Closed Littlebox performs the transformation in the center with arms, stamps, straps, paths, or a separate press/slot prop.

## Handoff Path

Use for workflows, customer journeys, publishing pipelines, sales handoffs, review loops, and assistant workflows.

Use one curved amber path. Littlebox carries, seals, stamps, or routes the object at the handoff point. Prefer a loose path over a straight process chart.

## Hidden Interior

Use for black boxes, model behavior, assumptions, hidden costs, and behind-the-scenes systems.

Littlebox stays closed. Represent hidden interior behavior externally with slips, tags, peephole labels, orbiting clues, or small outside props. A cutaway or visible interior fails this family.

## Pressure And Overflow

Use for bottlenecks, too many tools, overloaded context, queue problems, attention debt, or weak systems.

Littlebox stays closed while bracing, tying, blocking, catching, or carrying an external pile. Use coral only for the risk point. The lid stays closed under pressure.

## Evidence Bundle

Use for trust, proof, documentation, decision records, and validation.

Littlebox gathers evidence slips with side-seam twig arms and seals them into one usable external package. Coral can mark the final stamp.

## Mini Comic

Use for failure-to-fix sequences and dry operational humor.

Use 2-3 loose panels maximum. Each panel shows one action. Keep Littlebox consistent and closed in every panel.

## Metaphor Invention Rule

Convert the abstract idea into a physical box action:

- scattered inputs -> gather into an external stack beside closed Littlebox
- unclear decision -> sort pieces into labeled piles with twig arms
- fragile trust -> seal an evidence package
- broken handoff -> a sealed package snags on a curved path
- over-automation -> closed Littlebox braces an external spill of tiny tools
- context compression -> Littlebox folds a large sheet into a small parcel
- publishing -> Littlebox stamps or hands off one clean signal

Add one low-tech object only when it clarifies the idea: slip, tape, stamp, shelf, path, slot, scale, tag, latch, handle, tiny stack, or parcel.

## Spatial Rule

Distribute elements across both axes. A wide image should have one enlarged subject plus small supporting elements at different heights.

## Flow Mark Rule

- Use amber `#F59E0B` for flow paths, motion curves, and arrows.
- Keep flow marks thinner or lighter than Littlebox's black outline.
- Keep flow paths visually separate from the amber seam tape.
- Prefer one curved path over multiple straight arrows.
- Use black arrows only as tiny hand motion marks.

## Anti-Repeat Rule

Vary Littlebox actions across a set: sealing, stamping, carrying, sorting external slips, tying a bundle, blocking a path, handing off a parcel, folding a sheet, bracing a pile, or redirecting a curved path.

Keep the canonical front-left three-quarter box readable. Change surrounding objects before deforming Littlebox.
