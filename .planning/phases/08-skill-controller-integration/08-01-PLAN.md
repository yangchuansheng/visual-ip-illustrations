<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user request to route selection | User-provided aliases determine which route group and protected-character status apply. |
| controller to route-local references | `SKILL.md` decides which references are loaded and can accidentally leak Tom behavior into Xiaohei or Littlebox. |
| Tom route to host image generation | Protected-character prompt wording crosses into the image generator and must preserve route status and source-recreation gates. |
| generated output to delivery report | Delivery wording can imply broader availability or drop the rights-status note. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01-01 | Spoofing | route selection in `SKILL.md` | mitigate | Add exact Tom aliases and route id `tom`; preserve omitted-IP Xiaohei default. |
| T-08-01-02 | Tampering | mixed route groups in `SKILL.md` | mitigate | Require separate route groups and route-local references for Xiaohei, Littlebox, and Tom. |
| T-08-01-03 | Information Disclosure | Tom delivery block | mitigate | Keep `gated-authorized` and `references/ips/tom/rights.md` visible in Tom planning, generation, QA/edit, and delivery context. |
| T-08-01-04 | Elevation of Privilege | Tom generation routing | mitigate | Route Tom through protected-route QA and block source-frame recreation, show logos, title cards, default Jerry usage, broad cast expansion, generic-cat drift, excessive text, and route leakage. |
| T-08-01-05 | Repudiation | output path markers | mitigate | Add raw and escaped Tom path markers for later validation continuity. |
| T-08-01-SC | Tampering | package installs | accept | Phase 8 installs no npm, pip, or cargo packages. |
</threat_model>
