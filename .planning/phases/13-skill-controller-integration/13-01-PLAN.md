## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User prompt to route selection | Untrusted prompt text determines whether Xiaohei, Littlebox, Tom, Ferris, or mixed route groups are selected. |
| Route metadata to reference loading | `routing.md` required references determine which route-local policy files enter controller context. |
| Controller prompt assembly to image generation | `SKILL.md` instructions shape host-provided image generation prompts and repair paths. |
| Generated assets to delivery report | Save paths and route notes tell users where images were written and which route constraints were applied. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | Spoofing | Ferris route selection aliases | mitigate | Lock the six Ferris aliases, preserve `default=false`, and grep both `SKILL.md` and `routing.md`. |
| T-13-02 | Tampering | Ferris `required_references` | mitigate | Expand Ferris route metadata to the exact seven-file pack and assert every path. |
| T-13-03 | Repudiation | Ferris source/trademark context | mitigate | Keep `source-reviewed`, `references/ips/ferris/source.md`, and compact source/trademark note visible in planning, generation, QA/edit, and delivery. |
| T-13-04 | Information Disclosure | Route-local reference loading | mitigate | Require each selected route group to load only its own `required_references`. |
| T-13-05 | Denial of Service | Mixed-IP prompt blending | mitigate | Use one shared core idea with separate Xiaohei, Littlebox, Tom, and Ferris variant groups and output directories. |
| T-13-06 | Elevation of Privilege | Rust official-affiliation cues | mitigate | Carry Ferris QA failure categories and trademark-boundary edit gate into controller repair routing. |
