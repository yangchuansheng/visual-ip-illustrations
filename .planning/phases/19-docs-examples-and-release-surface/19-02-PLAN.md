## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Copyable prompts to skill runtime | Prompt examples influence route selection, reference loading, generated output paths, and maintainer smoke expectations. |
| Prompt examples to public sample policy | Examples can imply public generated asset availability if sample gates are missing. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-02-S | Spoofing | Sealos prompt route identity | mitigate | Require exact aliases, selected IP `Sealos Seal`, route status `brand-owned`, and source authority in smoke prompt. |
| T-19-02-T | Tampering | Mixed-IP route isolation | mitigate | Require each mixed route group to load only route-local references and keep route-specific status/source/rights/brand notes. |
| T-19-02-R | Repudiation | Smoke prompt expected fields | mitigate | Text-only smoke prompt records explicit expected selected IP, paths, references, markers, and public sample gate. |
| T-19-02-I | Information Disclosure | Prompt examples | accept | Examples expose route text already present in public route docs and no private generated sample asset. |
| T-19-02-D | Denial of Service | Runtime package | accept | Documentation-only prompt examples do not alter runtime loading. |
| T-19-02-E | Elevation of Privilege | Default route semantics | mitigate | Mixed prompts keep Sealos explicit and preserve omitted-IP Xiaohei behavior by route group separation. |
| T-19-02-SC | Tampering | npm/pip/cargo installs | accept | No package-manager installs are planned. |
</threat_model>
