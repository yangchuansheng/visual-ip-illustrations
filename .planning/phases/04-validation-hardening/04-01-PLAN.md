<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Repository files -> validator parser | Markdown/YAML/prose files are treated as untrusted text input to a local command. |
| Route metadata -> filesystem paths | `required_references` values cross from prose metadata into filesystem resolution. |
| Validator results -> maintainer action | PASS/FAIL logs guide release and documentation edits. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | Route reference path helpers | mitigate | Resolve route paths under the installable skill package and report escaping paths as failures before file checks. |
| T-04-02 | Repudiation | Validator logs | mitigate | Emit stable English check IDs, path-specific messages, and summary counts. |
| T-04-03 | Denial of Service | Validator runtime | mitigate | Use synchronous local file reads over the small repository file set and avoid network, browser, package-manager, and image-generation calls. |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No package-manager installs are introduced; the validator uses Node built-ins only. |
</threat_model>
